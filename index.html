Algorithm Reference

How each scheduling strategy works under the hood.

FCFS First Come First Serve

The simplest disk scheduling algorithm. Requests are serviced in the exact order they arrive in the queue. While fair and easy to implement, FCFS can result in large total head movements.

✓ Fair — No starvation
✗ Inefficient — No optimization
→ Best for: Light or ordered workloads
Pseudocode
for each request in queue (in order):
    move head to request
    record distance moved
SSTF Shortest Seek Time First

Selects the request closest to the current head position at each step. This greedy approach significantly reduces total movement, but requests far from the action can wait indefinitely — a problem called starvation.

✗ Starvation risk for distant requests
✓ Highly efficient — minimizes seeks
→ Best for: Throughput-heavy systems
Pseudocode
while pending requests exist:
    find request closest to head
    move head there
    record distance moved
SCAN Elevator Algorithm

The head moves in one direction, servicing all requests along the way, travels to the disk boundary, then reverses and services the other side. Like an elevator — bounded waiting time with good throughput.

✓ Bounded wait — no starvation
✓ High throughput
→ Best for: General-purpose scheduling
Pseudocode
move in chosen direction, service requests
when boundary reached: reverse direction
continue until all requests served
